# 🌍 My Travel App - Project State & Architecture

## 📌 О проекте
Приложение для планирования путешествий. Позволяет создавать поездки, управлять маршрутами по дням и времени, загружать документы, вести учет расходов и общаться в чате.

## 🛠 Технологический стек
- **Фреймворк:** Next.js (App Router)
- **Язык:** TypeScript
- **База данных:** SQLite + Prisma ORM
- **Стилизация:** Tailwind CSS + UI компоненты
- **Иконки:** Lucide React
- **ИИ-интеграция:** Gemini API (используется для умного импорта)
- **Хостинг / Инфраструктура:** Fly.io + Docker. База данных SQLite (`sqlite.db`) и загружаемые файлы (`uploads/`) сохраняются на подключенном постоянном диске Fly Volume (`voyami_data` на 3 ГБ, смонтированном в `/data`).

## ✅ Что уже реализовано

### 1. Управление поездками (Trips)
- Создание, редактирование и удаление поездок.
- Поддержка коллаборации: добавление участников (Trips Participants) с ролями (OWNER, MEMBER).

### 2. Маршруты и Таймлайн (Itinerary)
- Создание дней путешествия.
- Добавление активностей (События, Транспорт, Отели, Рестораны и т.д.) с привязкой ко времени.
- **Хронологическая сортировка:** Активности автоматически сортируются по времени в UI.
- **Умный импорт из Excel/Word (Smart Import):** 
  - Реализован парсинг файлов с помощью Gemini API.
  - Интерактивное превью перед сохранением.
  - При сохранении импорта старый маршрут полностью заменяется новым.

### 3. Места (Places)
- База интересных мест для каждой поездки.
- Категоризация (Ресторан, Кофе, Локация, Практическое).

### 4. Расходы (Expenses)
- Добавление расходов.
- Разделение расходов между участниками (кто платил и чья доля).

### 5. Документы (Documents)
- Загрузка файлов (билеты, брони отелей, страховки).
- Привязка документов к конкретным дням маршрута.
- **Персистентность файлов:** При сборке Docker-контейнера директория `public/uploads` автоматически привязывается к папке `/data/uploads` на постоянном диске, защищая загруженные файлы от удаления при рестартах.

### 6. Заметки и Чат
- Текстовые заметки для планирования.
- Внутренний чат поездки между участниками.

### 7. Интерактивная карта и Геолокация (Interactive Map)
- **Интеграция с Google Maps:** Отображение всех точек маршрута на карте с прорисовкой путей (Polylines) между ними.
- **Цветовое разделение по дням:** Каждому дню соответствует свой цвет маркеров и линий.
- **Интерактивная фильтрация:**
  - При выборе дня маркеры других дней затеняются (opacity + grayscale), а карта масштабируется под выбранный день.
  - При клике на маркер или элемент в боковой панели открывается всплывающее окно (InfoWindow) с деталями, а карта центрируется на месте.
- **Умное геокодирование и кэш:**
  - Автоматический поиск координат через Google Geocoder по английскому адресу `locationQuery` с фолбеком на название активности `title`.
  - Поиск ограничен страной путешествия (например, `AL` для Албании).
  - Фильтрация общих результатов: исключаются координаты уровня страны/области при неточном поиске.
  - Найденные координаты кэшируются в БД.
  - **Сброс устаревшего кэша:** При изменении названия или адреса геолокации в API координаты автоматически сбрасываются в `null`, вызывая повторный геокодинг.
- **Интерфейс геолокации:**
  - Добавлено поле **«Геолокация (на английском)»** в форму создания/редактирования активностей.
  - Точки без координат визуально выделяются в сайдбаре значком `❓` и бейджем `НЕ НА КАРТЕ`.

## ⚡️ Последние архитектурные исправления и фиксы

### 1. Кеширование в Next.js (Устранено)
- Для всех ключевых страниц (`/`, `/itinerary`, `/people`, `/expenses`, `/documents`, `/notes`, `/settings`) и API-роутов добавлена директива `export const dynamic = 'force-dynamic'`. Это решило проблему статического кеширования Next.js на хостинге, из-за которой добавление/удаление данных не отображалось без ручной перезагрузки.

### 2. Ошибки загрузки файлов (Устранено)
- Избавились от хардкода «Алина» при загрузке документов на бэкенде. Теперь Server Action `uploadDocumentAction` считывает ID текущего активного пользователя из куки (через cookies-хелпер) и связывает файлы с реальным автором. При отсутствии сессии используется фолбек на первого созданного участника.

### 3. Полностью динамический Дашборд
- Все захардкоженные заглушки на главной странице заменены на реальные данные из БД:
  - Сумма потраченных средств рассчитывается динамически.
  - Раздел «Следующий шаг» автоматически выводит ближайшее запланированное событие из бд.
  - Счётчик документов отражает реальное количество загруженных файлов.
  - Заглушка «3 пары» в Hero-карте заменена на динамическую страну поездки.

### 4. Починены все неактивные кнопки и UX-баги
- **Модалка заметок:** Браузерный диалог `prompt()` заменён на красивое всплывающее React-окно в стиле проекта.
- **Динамический чат:** Отправка сообщений и логика `isOwn` (подсветка собственных сообщений) привязаны к активному профилю из `useIdentity()`.
- **Кнопка темы:** Выбранная тема (тёмная/светлая) сохраняется в `localStorage` и применяется на клиенте при монтировании.
- **Места:** Кнопка «Карта» теперь перенаправляет на `/map`, удалён дублирующий холостой `useEffect` с лишними сетевыми запросами.
- **Документы:** Кнопка скачивания на карточках документов привязана к реальному пути файла. Добавлена работающая кнопка «Поделиться» (использует Web Share API или копирует ссылку в буфер).

## 🗄 Структура Базы Данных (Prisma)
- `User`: Пользователи системы.
- `Trip`: Основная сущность поездки.
- `TripParticipant`: Связь пользователей с поездками.
- `Day`: Дни маршрута (каскадно удаляются при удалении поездки).
- `Activity`: Активности внутри дня (каскадно удаляются при удалении дня, добавлены `latitude`, `longitude`, `locationQuery`).
- `Place`: Интересные места.
- `Document`: Загруженные файлы.
- `Expense` & `ExpenseParticipant`: Финансы и долги.
- `ChatMessage` & `Note`: Коммуникация и записи.

## 🚀 Как работать с ИИ в новых диалогах
Если контекст диалога стал слишком длинным или начал тормозить:
1. Смело открывай **новый диалог**.
2. Первым сообщением напиши: *"Привет! Прочитай файл `PROJECT_STATE.md` в корне проекта, чтобы понять текущий контекст. Наша следующая задача..."*
3. ИИ мгновенно погрузится в проект и продолжит работу без ошибок.
