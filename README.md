# 🌍 World Travel Hub

Совместное приложение для планирования путешествий. Позволяет создавать поездки, управлять маршрутами по дням, просматривать интерактивную карту с геолокацией, загружать документы, вести учет общих расходов и общаться в чате. Оптимизировано для мобильных телефонов (PWA) и совместного планирования с друзьями.

---

## 🛠 Технологический стек
- **Фреймворк:** Next.js (App Router)
- **База данных:** SQLite + Prisma ORM
- **Стилизация:** Tailwind CSS
- **Иконки:** Lucide React
- **ИИ-интеграция:** Gemini API (для умного импорта из Excel/Word)
- **Карты:** Google Maps API (с геокодированием и кэшированием координат)
- **Мобильный доступ:** PWA (Progressive Web App) + Service Worker (офлайн-режим)
- **Мультиплеер:** Динамическая смена профиля на базе Cookies

## 🚀 Локальный запуск и тест с телефона

### 1. Установка зависимостей и БД
```bash
# Установка пакетов
npm install

# Применение миграций БД и наполнение демонстрационными данными (сид)
npx prisma db push
npx prisma db seed
```

### 2. Запуск в домашней сети (для теста на смартфоне)
Чтобы открыть приложение на телефоне, подключенном к той же Wi-Fi сети:
1. Узнайте локальный IP-адрес компьютера (команда `ipconfig` в терминале Windows, ищите `IPv4-адрес`, например `192.168.1.50`).
2. Запустите dev-сервер на всех сетевых интерфейсах (`0.0.0.0`):
   ```bash
   npm run dev -- -H 0.0.0.0
   ```
3. Откройте в браузере телефона адрес: `http://<ваш-ip-адрес>:3000` (например, `http://192.168.1.50:3000`).

## 🚂 Деплой на Railway (Постоянный доступ 24/7)

Так как проект использует базу данных SQLite (файл `dev.db`), обычный деплой на Vercel не сохранит данные (Serverless-функции стирают локальные файлы при перезапуске). 

Для сохранения данных нужно развернуть проект на **Railway** с подключением постоянного диска (Volume).

### Шаг 1: Подготовка
1. Закоммитьте код и загрузите его в свой GitHub-репозиторий.
2. Зарегистрируйтесь на [Railway.app](https://railway.app).

### Шаг 2: Создание проекта в Railway
1. Нажмите **New Project** -> **Deploy from GitHub repo** и выберите свой репозиторий.
2. В настройках переменной окружения (**Variables**) добавьте:
   - `DATABASE_URL` = `file:/app/data/dev.db` (база данных будет лежать на постоянном диске)
   - `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` = *(ваш ключ Google Maps)*
   - `GEMINI_API_KEY` = *(ваш ключ Gemini)*
   - `PORT` = `3000`

### Шаг 3: Подключение постоянного диска (Volume)
Для того чтобы база данных SQLite не стиралась при каждом обновлении кода:
1. Перейдите во вкладку **Settings** вашего сервиса в Railway.
2. Найдите раздел **Volumes** и нажмите **Add Volume**.
3. Назовите диск (например, `travel-data`) и укажите точку монтирования (Mount Path):
   ```
   /app/data
   ```
4. При запуске приложения Prisma автоматически создаст там файл базы данных `dev.db`.

### Шаг 4: Настройка сборки и запуска
Убедитесь, что Railway использует правильные команды сборки. Вкладка **Settings** -> **Deploy**:
- **Build Command:** `npx prisma generate && next build`
- **Start Command:** `npx prisma db push && next start` *(это автоматически создаст таблицы в базе данных на диске при первом запуске)*

После завершения деплоя Railway выдаст вам публичную красивую ссылку (например, `https://travel-app.up.railway.app`), по которой вы и ваши друзья сможете заходить с компьютеров и телефонов в любое время!
